OS Detection Module¶

Module containing operating system detection functionality. This module takes a list of IP addresses to run OS detection on. There are three tests that are used to determine the OS.

One is the TTL value. Windows, by default, will send packets with a starting TTL value of 128. Other systems will send packets with a starting TTL value of 64.

The IPID is tested with a sequence of tests. The program will check if unprompted SYN/ACKs receive RSTs with IPIDs of 0. Next, the program will test if the TCP IPID field consistently increments by 1. If the IPID is incrementing by 1, it is then checked for per-connection behavior. If the IPID did not increment by 1, The IPID is also checked for ‘random’ behavior. If each IPID is consistently greater than the one preceding it, then it is determined to be ‘not random’. If the IPIDs break this chain of sequence, then they are considered random.

os_detection.check_for_increment_by_one_ipids(target: str) → bool | None¶

Checks if the IPIDs from a specified target are incrementing by 1.

Parameters:

target (str) – The IP address to send packets to.

Returns:

Whether the IPID value seems to increment by 1. If no ICMP response was received, None is returned.

Return type:

bool | None

os_detection.check_for_incremental_per_connection(target: str) → bool | None¶

Determine if the IPID increment is per-connection.

This will only run if the IPIDs are previously found to be globally incrementing. If so, this test will check if the IPID seems to be incrementing on a shared (bucket or global) counter, or if the IPID is specific to each connection.

Parameters:

target (str) – The IP address to send packets to.

Returns:

Whether the device has a per-connection IPID. If no ICMP response was received, None is returned.

Return type:

bool | None

os_detection.check_for_random_ipid(target: str) → bool | None¶

Evaluate if the IPID seems to be random or incrementing on some range.

This test will check if the IPID values seem to be ‘random’. This is done by sending a stream of traffic and seeing if the IPIDs are consistently increasing by some value.

Parameters:

target (str) – The IP address to send packets to.

Returns:

Whether the device appears to have a random IPID. If no ICMP response was received, None is returned.

Return type:

bool | None

os_detection.send_synack(target: str) → bool | None¶

Sends unprompted SYN/ACK traffic and checks if the IPID is zero on RST.

Parameters:

target (str) – The IP address to send packets to.

Returns:

Whether the IPIDs are zero. Returns None if no response was received for the sent SYN/ACK packets.

bool | None
