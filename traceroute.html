Traceroute Module¶

Module containing traceroute functionality.

The trace function can perform either a TCP or ICMP traceroute to a list of user specified target hosts. The TCP traceroute works by sending a TCP SYN packet to each hop, until either a SYN/ACK or RST packet is received from the destination host. A SYN/ACK response indicates that the destination host’s target port is open, and a RST response means the destination host’s target port is closed. The ICMP traceroute works by sending ICMP Echo Request packets to each hop until an ICMP Echo Reply packet is received from the destination host. At each hop, the IP address of the hop and the time it took to reply to each of the three probe packets is printed to the screen. Additionally, the user can specify whether they want an image created, which visualizes the traceroute operations that took place.

Typical usage example:

trace((“1.1.1.1”, “8.8.8.8”), TraceType.TCP, 30, 0.5)

traceroute._build_graph(graph: Graph, hops: list[str]) → None¶

Helper method that builds a graph based on the traceroute results.

  • graph (nx.Graph) – A graph object to add the traversed hops to.

  • hops (list[str]) – A list of hops that were traversed on the path to the destination host.

traceroute._print_hop_info(hop_num: int, ip: str, response_times: list[str], destination: bool) → None¶

Helper method that prints out information pertaining to the current hop.

This method will print the hop number, IP address of the hop, and three times in milliseconds. Each of the three times represent the round-trip times for each of the three probe packets sent to the hop.

Parameters:

  • hop_num (int) – The number of the current host on the path to the destination host.

  • ip (str) – The IP address of the current host.

  • response_times (list[str]) – A list of how long it took the host to respond to each of the three probe packets that were sent.

  • destination (bool) – Boolean representing whether the current host is the destination host or not.

traceroute._save_graph_image(graph: Graph) → None¶

Helper method that draws the provided graph and saves it as a png.

The visualization of the traceroute operations will be saved as a png named traceroute.png, in the local directory where the program was executed.

Parameters:

graph (nx.Graph) – A graph object that visualizes the performed traceroute operations.

traceroute._send_probes(target: str, trace_type: TraceType, ttl: int, response_times: list[str], timeout) → Packet | None¶

Helper method that sends three probe packets to the next hop.

Parameters:

  • target (str) – The final destination IP address that we are tracing a route to.

  • trace_type (TraceType) – The type of trace to perform. Determines the type of packets that will be sent to the next hop.

  • ttl (int) – The Time To Live value for the packets that will be sent to the next hop.

  • response_times (list[str]) – A list of round-trip times in ms for each probe packet that was sent to the hop.

  • timeout (float) – Timeout in seconds for how long to wait to receive a response packet from a host.

traceroute.trace(targets: tuple[str, ...], trace_type: TraceType, max_hops: int, timeout: float, create_image: bool) → None¶

Prints out a trace of the route taken to each target host.

Parameters:

  • targets (tuple[str, ...]) – A tuple of target IP addresses to trace.

  • trace_type (TraceType) – The type of trace to perform.

  • max_hops (int) – The number of hops to trace before ending the program.

  • timeout (float) – Timeout in seconds for how long to wait to receive a response packet from a host.

  • create_image (bool) – Boolean value representing whether or not to create a visualization of the traceroute operations.
