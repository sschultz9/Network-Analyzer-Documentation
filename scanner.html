Scanner Module¶

Module containing the Scanner class for network scanning functionality.

The Scanner class can perform a variety of scans on a target network, which is specified via CIDR notation. If the scan type is a SYN scan, then the scan will enumerate a list of ports on each target host as specified by the user. Each port is sent as a SYN packet. If a SYN/ACK is received as a response, the user port is listed as open. If a RST is received as a response, the port is marked as closed. Otherwise, the port is listed as being filtered. If verbose is set to true in the scan configuration, then the scanning results for all ports will be displayed. Otherwise only the information pertaining to open ports will be displayed. Additionally, if service scan is set to true in the configuration, each open port will be further scanned to determine the service running on the open port, as well as the version of the service if it can be determined. Otherwise, a well-known mapping between ports and services will be utilized. Finally, if the cve scan option is set to true in the configuration then each service that is found to be running on an open port will be checked for associated CVEs, and the IDs for these CVEs will be written to an output file. The ICMP scan is much simpler as it just sends an ICMP Echo Request packet to every target host and lists the hosts that responded as being alive.

Typical usage example:

config = ScanConfig(
    targets=["192.168.1.0/24"],
    scan_type=ScanType.ICMP,
    ports=(22, 23, 80, 443),
    timeout=0.5,
    verbose=False,
    os_detection=True,
    service_scan=True,
    cve_scan=True
)
scanner = Scanner(config)
alive_hosts = scanner.scan()

If True, check detected services and versions against known CVEs.

Tuple of TCP ports to scan.

Type of network scan to perform (SYN or ICMP).

If True, attempt to identify services and versions on open ports.

List of target IP addresses to be scanned.

timeout: float¶

Timeout in seconds to wait for a response packet.

verbose: bool¶

If True, print all port results, otherwise only open ports.

_null_probe(target: str, port: int) → str | None¶

Try to grab a welcome banner from the open port via a NULL probe.

A NULL probe is when we create a connection with the target host on the open port and then listen for responses without sending any data. We do this because many services identify themselves via an initial welcome banner.

Parameters:

  • target (str) – The target host with the open port to grab the banner from.

  • port (int) – The open port to grab the banner from.

Returns:

The welcome banner that was retrieved from the open port, or None if no welcome banner could be retrieved.

Return type:

str | None
